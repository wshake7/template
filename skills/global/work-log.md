# AI 工作记录规范

用于约束 AI 以“里程碑”方式记录工作过程与结果，避免逐步流水账造成冗余。

## 记录文件

- 固定路径: `AI_WORK_LOG.md`
- Git 辅助记录: `AI_GIT_LOG.md`

## 何时记录

- 一个任务完成阶段性目标（里程碑）时
- 发生关键决策变更（方案变更、回滚、失败复盘）时
- 准备交付给用户前（最终总结）时

## 如何记录

按“少量高价值记录”原则写入，优先更新当前里程碑条目，不必每次小改都追加新条目。
在需要新增条目时使用以下结构：

```md
## [YYYY-MM-DD HH:mm] <任务标题>
- 目标: <本次任务目标>
- 改动文件:
  - <path1>
  - <path2>
- 执行命令:
  - <command1>
  - <command2>
- 结果: <通过/失败 + 简述>
- 风险/待办: <无/具体项>
- Git 依据: <对应 AI_GIT_LOG.md 的时间戳标题 / 复用上次提交记录>
```

## 记录要求

1. 只记录事实，不写模糊表述
2. 文件路径使用项目相对路径
3. 命令写可复现的完整命令
4. 若命令失败，必须写失败原因和是否已修复
5. 若无改动文件，明确写 `- 改动文件: 无`
6. 同一任务中的小步修改，优先“更新最近条目”而不是新增条目
7. 若用户手动改过 AI 产物，优先参考 `AI_GIT_LOG.md` 再续写工作记录
8. 若本次没有新提交，`Git 依据` 写 `复用上次提交记录`
