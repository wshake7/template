# AI 工作记录规范

## 场景与目标

- 适用场景：用户要求记录、需求状态变化、阶段性交付、失败复盘。
- 目标：用少量里程碑记录保留事实，不写逐步流水账。

## 记录文件

- 工作记录：`AI_WORK_LOG.md`
- Git 证据：`AI_GIT_LOG.md`

## 何时更新

1. 用户明确要求记录。
2. `AI_REQUIREMENTS.md` 中的需求状态、备注或验收信息需要同步。
3. 一个任务完成阶段性目标，需要留下交付事实。
4. 发生方案变更、失败复盘、回滚或用户手动修正 AI 产物。

## 记录模板

```md
## [YYYY-MM-DD HH:mm] <任务标题>
- 目标: <本次任务目标>
- 改动文件:
  - <path>
- 执行命令:
  - <command>
- 结果: <通过/失败 + 简述>
- 风险/待办: <无/具体项>
- Git 依据: <AI_GIT_LOG.md 对应标题 / 复用上次提交记录>
```

## 写法要求

1. 只记录事实和结果，不写心理活动。
2. 文件路径使用项目相对路径。
3. 命令写可复现的完整命令。
4. 命令失败时记录失败原因、是否已修复、是否等待用户决定。
5. 同一任务的小步调整优先更新最近条目，不频繁追加新条目。
6. 没有新提交时，`Git 依据` 写 `复用上次提交记录`。
