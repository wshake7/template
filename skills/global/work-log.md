# AI 工作记录规范

用于约束 AI 在每次实际改动后，将工作过程与结果写入固定记录文件。

## 记录文件

- 固定路径: `AI_WORK_LOG.md`
- Git 辅助记录: `AI_GIT_LOG.md`

## 何时记录

- 修改了任何源码、配置、脚本、CI、hooks、skills 文档
- 执行了关键命令并据此给出结论（如 lint/test/build）

## 如何记录

每次任务结束时，在 `AI_WORK_LOG.md` 追加一条记录，使用以下结构：

```md
## [YYYY-MM-DD HH:mm] <任务标题>
- 目标: <本次任务目标>
- 改动文件:
  - <path1>
  - <path2>
- 执行命令:
  - <command1>
  - <command2>
- 结果: <通过/失败 + 简述>
- 风险/待办: <无/具体项>
- Git 依据: <对应 AI_GIT_LOG.md 的时间戳标题 / 无（未采集）>
```

## 记录要求

1. 只记录事实，不写模糊表述
2. 文件路径使用项目相对路径
3. 命令写可复现的完整命令
4. 若命令失败，必须写失败原因和是否已修复
5. 若无改动文件，明确写 `- 改动文件: 无`
6. 若用户手动改过 AI 产物，优先参考 `AI_GIT_LOG.md` 再续写工作记录
