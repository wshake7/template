# 国际化 (i18next)

本项目使用 i18next + react-i18next 实现多语言支持。

## 目录结构

```
front/apps/admin/
├── locales/
│   └── zh.json          # 中文翻译文件
└── src/
   ├── i18n.ts           # i18n 初始化配置
   └── (各页面组件中使用)
```

## i18n 配置

位置: `src/i18n.ts`

配置包含:
- 语言检测 (i18next-browser-languagedetector)
- 翻译资源加载
- 命名空间配置
- fallback 语言配置
- 当前仅启用 `zh` 资源

## 翻译文件格式

位置: `locales/zh.json`

```json
{
  "menu": {
    "dashboard": "仪表盘",
    "account": "账户"
  },
  "button": {
    "submit": "提交",
    "cancel": "取消"
  }
}
```

## 组件中使用

### 基本使用

```typescript
import { useTranslation } from 'react-i18next'

function MyComponent() {
  const { t } = useTranslation()

  return (
    <div>
      <h1>{t('menu.dashboard')}</h1>
      <button>{t('button.submit')}</button>
    </div>
  )
}
```

### 带参数的翻译

```typescript
// locales/zh.json
{
  "greeting": "你好，{{name}}！"
}

// 组件中
const { t } = useTranslation()
<p>{t('greeting', { name: '张三' })}</p>
```

### 复数形式

```typescript
// locales/zh.json
{
  "itemCount": "{{count}} 个项目",
  "itemCount_other": "{{count}} 个项目"
}

// 组件中
<p>{t('itemCount', { count: 5 })}</p>
```

## 添加新语言

1. 在 `locales/` 下创建新的翻译文件 (如 `en.json`)
2. 在 `i18n.ts` 中导入并配置:

```typescript
import en from '@/locales/en.json'

const resources = {
  zh: { translation: zh },
  en: { translation: en },
}

// 添加到 i18n.init() 配置中，并补充 preload
```

## 语言切换

默认通过浏览器语言检测；检测不到时回退到 `fallbackLng: 'zh'`。

## 注意事项

1. 翻译 key 应使用点号分隔的层级结构
2. 避免在代码中硬编码翻译文本
3. 组件卸载时不需要手动销毁 i18n 实例
