# Admin 国际化

## 场景与目标

- 适用场景：新增或调整 Admin 文案、翻译 key、语言资源。
- 目标：保持 i18next 配置和资源文件一致，避免散落硬编码。

## 目录/文件位置

- 初始化：`front/apps/admin-react/src/i18n.ts`
- 中文资源：`front/apps/admin-react/locales/zh.json`
- 页面和组件：`front/apps/admin-react/src/**`

## 当前约定

- 当前默认语言资源为 `zh`。
- 翻译 key 使用点号层级，如 `menu.dashboard`。
- 组件中通过 `useTranslation()` 获取 `t`。
- 普通后台文案目前以中文为主；只有涉及多语言维护时才必须入资源文件。

## 使用示例

```typescript
import { useTranslation } from 'react-i18next'

function PageTitle() {
  const { t } = useTranslation()
  return <h1>{t('menu.dashboard')}</h1>
}
```

## 新增语言步骤

1. 在 `locales/` 下新增资源文件，如 `en.json`。
2. 在 `src/i18n.ts` 导入资源并加入 `resources`。
3. 检查 fallback、preload 和语言检测逻辑。
4. 补齐已有 key，避免运行时显示 key 本身。

## 注意事项

1. key 命名按模块聚合，不把长句随意铺在顶层。
2. 带变量文案使用 i18next 插值，如 `{{name}}`。
3. 删除或改名 key 前先 `rg` 全局查引用。
4. i18next API 用法问题必须按 `AGENTS.md` 使用 Context7。
