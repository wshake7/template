# Skill: Memory Maintenance

## 何时使用

当任务完成、需要跨对话保存状态、更新任务列表、整理技能库，或用户明确要求记录项目记忆时使用。

## 记忆文件

```text
AGENTS.md                 # 助手身份、规则和工作方式
MEMORY.md                 # 长期记忆：偏好、技术栈、持久事实
memory/YYYY-MM-DD.md      # 每日日志：当天做了什么、决策、临时笔记
memory/tasks.md           # 跨对话任务追踪
.agents/skills/**         # 项目级可复用技能
```

## 更新原则

- `MEMORY.md` 只写长期有用的信息，不记录一次性细节。
- 每日日志记录“今天做了什么”和重要决策，保持短句。
- `memory/tasks.md` 适合较大任务或跨对话任务；小修小补无需制造任务噪音。
- 新增或改进技能时，同步更新 `.agents/skills/README.md`。
- 不要为了记录而改动无关文件。

## 任务状态格式

```markdown
- [ ] 待办事项
- [~] 进行中的事项
- [x] 已完成事项
```

## 收尾流程

1. 检查 `git status --short`，区分本次改动和用户已有改动。
2. 若任务有长期价值，更新 `memory/YYYY-MM-DD.md`。
3. 若任务已在 `memory/tasks.md` 中登记，更新状态；如是新大型任务，可新增条目。
4. 若新增技能，更新 `.agents/skills/README.md` 索引。
5. 按变更范围执行验证命令。
6. 最终回复中简要说明改了哪些文件、验证了什么、哪些验证未执行。

## 提交注意

- 仓库规则鼓励任务结束后提交变更，但遇到大量用户未提交改动时，只能提交本次明确相关文件。
- 未经用户明确要求，不要把无关脏文件一起 stage 或提交。
- 如需提交，先用 `git diff -- <path>` 复核自己的改动，再精确 `git add <path...>`。
